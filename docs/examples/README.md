# ScreenForge Examples

Real, committed examples generated by driving ScreenForge against live public sites.
Each pair is a **workflow you describe** (`*.workflow.yaml`) plus the **pytest file ScreenForge generated from it** (`test_*.py`) — not hand-written. Each generated test replays green.

## How these run

These examples live OUTSIDE `pytest.ini`'s `testpaths`, so they are **never collected by default CI**.
Regenerate one from its workflow:

```bash
.venv/bin/python agent_cli.py --workflow docs/examples/web/inputs_form.workflow.yaml --platform web
```

Or replay the committed generated test against a live browser:

```bash
RUN_LIVE_PLATFORM_TESTS=true TEST_PLATFORM=web \
  .venv/bin/python -m pytest docs/examples/web/test_inputs_form.py -q
```

| Workflow | Generated test | Flow | What it proves |
|---|---|---|---|
| `web/inputs_form.workflow.yaml` | `web/test_inputs_form.py` | Open a form page, type into a field, assert it | Describe-in-YAML → replayable multi-step pytest |
| `web/checkboxes.workflow.yaml` | `web/test_checkboxes.py` | Open checkboxes page, toggle, assert state | Stateful single-page interaction; self-heal substrate |

See `web/SELFHEAL.md` for the self-healing walkthrough.
